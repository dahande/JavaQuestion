Java Silver 総合問題 - Part 8

問題61

次のJavaコードをコンパイルするとどうなりますか？


public class Test {
    public static void main(String[] args) {
        int x = 10;
        int y = 0;
        System.out.println(x / y);
    }
}
正解はA) ArithmeticExceptionが発生します。
解説: yが0のため、x / y はArithmeticExceptionをスローします。そのため、このコードは実行時エラーとなります。

問題62

次のJavaコードをコンパイルするとどうなりますか？


public class Test {
    public static void main(String[] args) {
        String str1 = "Java";
        String str2 = new String("Java");
        System.out.println(str1 == str2);
    }
}
正解はB) falseです。
解説: str1はStringプールから参照され、str2は新しいStringオブジェクトを生成しています。== 演算子は参照の比較を行うため、str1とstr2は異なるオブジェクトを参照しているため false が出力されます。

問題63

次のJavaコードをコンパイルするとどうなりますか？


public class Test {
    public static void main(String[] args) {
        int i = 1;
        switch (i) {
            case 1:
                System.out.println("One");
            case 2:
                System.out.println("Two");
            default:
                System.out.println("Default");
        }
    }
}
正解はD) One Two Defaultです。
解説: switch文でiが1と一致するため、case 1のブロックが実行されます。その後、case 2のラベルがないため、次のdefaultブロックも実行されます。したがって、"One Two Default"が出力されます。

問題64

次のJavaコードをコンパイルするとどうなりますか？


public class Test {
    public static void main(String[] args) {
        int x = 5;
        int y = x++ * 2 + x--;
        System.out.println("x = " + x + ", y = " + y);
    }
}
正解はA) x = 5, y = 12です。
解説: x++ * 2 + x-- は後置インクリメントと後置デクリメントを含みます。xはまず5で初期化され、yの計算中にxが6に増加し、それが2倍されて12になります。その後、xがデクリメントされ、最終的にx = 5となります。

問題65

次のJavaコードをコンパイルするとどうなりますか？


public class Test {
    public static void main(String[] args) {
        int x = 0;
        while (x < 5) {
            System.out.print(x + " ");
            x += 2;
        }
    }
}
正解はC) 0 2 4です。
解説: whileループはxが5未満の間繰り返されます。xは0から始まり、2ずつ増加していきます。したがって、"0 2 4 "が出力されます。

問題66

次のJavaコードを実行した結果、何が出力されますか？


public class Test {
    public static void main(String[] args) {
        try {
            throw new Exception();
        } catch (Exception e) {
            throw new RuntimeException();
        } finally {
            System.out.println("Finally block");
        }
    }
}
正解はD) Finally blockが出力されて、RuntimeExceptionがスローされます。
解説: tryブロックでExceptionがスローされ、catchブロックでRuntimeExceptionがスローされます。finallyブロックは必ず実行されます。

問題67

次のJavaコードを実行した結果、何が出力されますか？


public interface MyInterface {
    void performAction();
}

public class Test {
    public static void main(String[] args) {
        MyInterface action = () -> System.out.println("Performing action");
        action.performAction();
    }
}
正解はB) Performing actionが出力されます。
解説: MyInterfaceをラムダ式で実装し、performActionメソッドをオーバーライドしています。このラムダ式は"Performing action"を出力します。

問題68

次のJavaコードを実行した場合、スレッドセーフですか？


public class Test {
    private static int count = 0;

    public static synchronized void increment() {
        count++;
    }

    public static void main(String[] args) throws InterruptedException {
        Thread thread1 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                increment();
            }
        });

        Thread thread2 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                increment();
            }
        });

        thread1.start();
        thread2.start();
        thread1.join();
        thread2.join();

        System.out.println("Count: " + count);
    }
}
正解はA) スレッドセーフではありません。
解説: incrementメソッドはsynchronizedで修飾されていますが、このコードはスレッドセーフではありません。複数のスレッドが同時にincrementメソッドを呼び出す可能性があり、結果としてcountの値が正しくないことがあります。

問題69:

次のJavaコードを実行した結果、何が出力されますか？


import java.util.ArrayList;
import java.util.List;

public class Test {
    public static void main(String[] args) {
        List<? extends Number> list = new ArrayList<Integer>();
        list.add(null);
        System.out.println(list.size());
    }
}
正解はC) コンパイルエラーが発生します。
解説: は「Numberまたはそのサブクラス型のリスト」を表しますが、この型のリストにはnull以外の要素を追加することができません。そのため、コンパイルエラーが発生します。

問題70

次のJavaコードを実行した結果、何が出力されますか？


public class Test {
    public static void main(String[] args) {
        Test test = null;
        test.staticMethod();
    }

    public static void staticMethod() {
        System.out.println("Static method called");
    }

    public void instanceMethod() {
        System.out.println("Instance method called");
    }
}
正解はD) NullPointerExceptionが発生します。
解説: staticMethodは静的メソッドであり、インスタンスの生成なしに呼び出すことができます。ただし、インスタンスメソッドinstanceMethodはインスタンスが存在しないと呼び出すことができません。そのため、このコードはNullPointerExceptionをスローします。
