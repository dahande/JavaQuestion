オブジェクト指向プログラミング（OOP）入門

オブジェクト指向とは？

オブジェクト指向プログラミング（OOP）は、ソフトウェア設計の一つの方法で、データをオブジェクトとして扱い、それらを組み合わせてプログラムを構築します。オブジェクトはクラスのインスタンスとして生成され、データとメソッドを含みます。

クラスとオブジェクト

クラスはオブジェクトの設計図です。クラスには属性（データ）とメソッド（関数）が定義され、これに基づいてオブジェクトが生成されます。以下は、クラスとオブジェクトの関係を示す簡単な例です。

// クラスの定義
public class Car {
    String brand;
    int speed;

    // コンストラクタ
    public Car(String brand, int speed) {
        this.brand = brand;
        this.speed = speed;
    }

    // メソッド
    public void displayInfo() {
        System.out.println("ブランド: " + brand + ", 速度: " + speed + " km/h");
    }
}

// メインクラスの定義
public class Main {
    public static void main(String[] args) {
        // インスタンスの生成
        Car car1 = new Car("トヨタ", 120);
        car1.displayInfo(); // 出力: ブランド: トヨタ, 速度: 120 km/h

        // 別のインスタンスを生成
        Car car2 = new Car("ホンダ", 100);
        car2.displayInfo(); // 出力: ブランド: ホンダ, 速度: 100 km/h
    }
}

オブジェクト指向の主要な原則

OOPには3つの主要な原則があります：

  • カプセル化（Encapsulation）: データを隠蔽し、データへのアクセスを制御するメソッドを提供します。
  • 継承（Inheritance）: 既存のクラス（親クラス）を基に新しいクラス（子クラス）を作成し、親クラスの属性とメソッドを再利用します。
  • ポリモーフィズム（Polymorphism）: 同じメソッド名で異なる動作を実装できるようにします。

カプセル化の例

// カプセル化の例
public class Person {
    private String name;
    private int age;

    // コンストラクタ
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // ゲッターとセッター
    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }
}

この例では、`name` と `age` フィールドはプライベートに設定され、直接アクセスできません。これにより、データの整合性が保たれます。データにアクセスするために、ゲッターとセッターが提供されています。

継承の例

// 継承の例
// 親クラス
public class Animal {
    public void eat() {
        System.out.println("食べる");
    }
}

// 子クラス
public class Dog extends Animal {
    public void bark() {
        System.out.println("吠える");
    }
}

// メインクラス
public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.eat(); // 出力: 食べる
        dog.bark(); // 出力: 吠える
    }
}

この例では、`Dog` クラスは `Animal` クラスを継承しています。`Dog` クラスは `Animal` クラスの `eat` メソッドを再利用でき、さらに `bark` メソッドを追加しています。

ポリモーフィズムの例

// ポリモーフィズムの例
public class Animal {
    public void makeSound() {
        System.out.println("動物の音");
    }
}

public class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("ワンワン");
    }
}

public class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("ニャーニャー");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myDog = new Dog();
        Animal myCat = new Cat();
        myDog.makeSound(); // 出力: ワンワン
        myCat.makeSound(); // 出力: ニャーニャー
    }
}

この例では、`Animal` クラスに `makeSound` メソッドが定義されており、`Dog` クラスと `Cat` クラスでそれぞれオーバーライドされています。これにより、同じ `makeSound` メソッドが異なる動作を実行します。
