Java 例外クラスの解説

ArrayIndexOutOfBoundsException

ArrayIndexOutOfBoundsExceptionは、配列の範囲外のインデックスにアクセスしようとしたときにスローされる例外です。例えば、次のようなコードが原因で発生します：


public class Main {
    public static void main(String[] args) {
        int[] array = {1, 2, 3};
        System.out.println(array[3]); // 有効なインデックス範囲外なので例外がスローされる
    }
}

この例外を処理するには、try-catchブロックを使用します：


public class Main {
    public static void main(String[] args) {
        int[] array = {1, 2, 3};
        try {
            System.out.println(array[3]);
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("配列の範囲外アクセスが発生しました: " + e.getMessage());
        }
    }
}

NullPointerException

NullPointerExceptionは、null参照を使用してオブジェクトのメソッドやプロパティにアクセスしようとしたときにスローされる例外です。例えば、次のようなコードが原因で発生します：


public class Main {
    public static void main(String[] args) {
        String str = null;
        System.out.println(str.length()); // null参照でメソッドを呼び出そうとするので例外がスローされる
    }
}

この例外を処理するには、try-catchブロックを使用します：


public class Main {
    public static void main(String[] args) {
        String str = null;
        try {
            System.out.println(str.length());
        } catch (NullPointerException e) {
            System.out.println("NullPointerExceptionが発生しました: " + e.getMessage());
        }
    }
}

ArithmeticException

ArithmeticExceptionは、数値計算のエラー（例えば、ゼロ除算）によってスローされる例外です。例えば、次のようなコードが原因で発生します：


public class Main {
    public static void main(String[] args) {
        int result = 10 / 0; // ゼロ除算のため例外がスローされる
    }
}

この例外を処理するには、try-catchブロックを使用します：


public class Main {
    public static void main(String[] args) {
        try {
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            System.out.println("ArithmeticExceptionが発生しました: " + e.getMessage());
        }
    }
}

ClassCastException

ClassCastExceptionは、オブジェクトを不正なクラスにキャストしようとしたときにスローされる例外です。例えば、次のようなコードが原因で発生します：


public class Main {
    public static void main(String[] args) {
        Object obj = new String("Hello");
        Integer num = (Integer) obj; // 不正なキャストのため例外がスローされる
    }
}

この例外を処理するには、try-catchブロックを使用します：


public class Main {
    public static void main(String[] args) {
        Object obj = new String("Hello");
        try {
            Integer num = (Integer) obj;
        } catch (ClassCastException e) {
            System.out.println("ClassCastExceptionが発生しました: " + e.getMessage());
        }
    }
}

IllegalArgumentException

IllegalArgumentExceptionは、不正な引数がメソッドに渡されたときにスローされる例外です。例えば、次のようなコードが原因で発生します：


public class Main {
    public static void main(String[] args) {
        Thread t = new Thread();
        t.setPriority(11); // 不正なスレッド優先度のため例外がスローされる
    }
}

この例外を処理するには、try-catchブロックを使用します：


public class Main {
    public static void main(String[] args) {
        Thread t = new Thread();
        try {
            t.setPriority(11);
        } catch (IllegalArgumentException e) {
            System.out.println("IllegalArgumentExceptionが発生しました: " + e.getMessage());
        }
    }
}
