Java モジュールの基礎知識

モジュールとは

Javaモジュールは、Java 9で導入された機能で、プログラムをモジュール単位で管理するための仕組みです。モジュールを使うことで、大規模なアプリケーションを小さな部分に分割し、それぞれの部分が明確に定義されたインターフェースを通じて他の部分と通信できます。

モジュールの構成要素

モジュールの基本的な構成要素は以下の通りです：

  • モジュール宣言ファイル (module-info.java): モジュールの名前や依存関係を定義します。
  • パッケージ: モジュール内に含まれるクラスやインターフェースをグループ化します。

モジュールの宣言

モジュール宣言は、モジュールの名前と、そのモジュールが依存する他のモジュールを指定します。以下は基本的なモジュール宣言の例です：

module mymodule {
    requires java.base;
    exports com.example.mypackage;
}

この例では、mymoduleというモジュールを宣言し、java.baseモジュールに依存していることを示しています。また、com.example.mypackageパッケージを他のモジュールに公開しています。

モジュールの依存関係

モジュールは、requiresキーワードを使用して他のモジュールに依存することを宣言できます。これにより、依存モジュールの公開されたパッケージを使用することができます。

module mymodule {
    requires anothermodule;
}

この例では、mymoduleがanothermoduleに依存していることを示しています。

モジュールのエクスポート

モジュールは、exportsキーワードを使用して特定のパッケージを他のモジュールに公開することができます。公開されたパッケージ内のクラスやインターフェースは、他のモジュールからアクセス可能です。

module mymodule {
    exports com.example.mypackage;
}

この例では、com.example.mypackageパッケージが他のモジュールに公開されています。

モジュールの使用例

以下に、モジュールを使用した簡単なJavaプロジェクトの例を示します：

1. モジュール宣言ファイル

// module-info.java
module myapp {
    requires java.sql;
    exports com.example.myapp;
}

2. パッケージとクラスの定義

// src/com/example/myapp/Main.java
package com.example.myapp;

public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, Java Modules!");
    }
}

このプロジェクトでは、myappモジュールがjava.sqlモジュールに依存し、com.example.myappパッケージを公開しています。
