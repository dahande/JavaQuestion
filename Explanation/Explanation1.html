Javaのインタフェース

Javaでインタフェース（interface）とは？

Javaにおけるインタフェースは、クラスの一種で、実装を持たないメソッドの集まりです。具体的な実装を持たず、メソッドのシグネチャ（宣言部分）だけを提供します。これにより、異なるクラスが同じインタフェースを実装することで、共通の振る舞いを保証します。

インタフェースの定義例

以下は、Javaでインタフェースを定義し、それを実装するクラスの例です。

// インタフェースの定義 public interface A { // デフォルトメソッドの定義 default void sample() { System.out.println("sample"); } } // インタフェースを実装するクラスの定義 public class B implements A { // インタフェースのメソッドをオーバーライドする必要はありません } // メインクラスの定義 public class Main { public static void main(String[] args) { // インタフェース型の変数にクラスのインスタンスを代入 A a = new B(); // インタフェースのメソッドを呼び出す a.sample(); // 出力: "sample" } }

インタフェースの利点

インタフェースを使用すると、以下のような利点があります。

  • 多重継承の代替: Javaでは単一継承しかサポートしていないため、複数のインタフェースを実装することで多重継承の効果を得ることができます。
  • 柔軟性と拡張性: インタフェースを実装するクラスは自由に切り替えることができ、プログラムの柔軟性と拡張性を高めます。
  • コードの再利用: 同じインタフェースを実装するクラスは同じ振る舞いを持ち、コードの再利用が容易になります。
