Javaの例外処理の解説

プログラムの紹介

このプログラムは、例外処理の流れを示しています。testメソッド内でNullPointerExceptionが発生し、それをキャッチして特定の処理を行う例です。

ソースコード


package mypack;

public class Main {
    public static void main(String[] args) {
        System.out.print(test(null));
    }

    private static String test(Object obj) {
        try {
            System.out.println(obj.toString());
        } catch (NullPointerException e) {
            return "A";
        } finally {
            System.out.println("B");
        }
        return "C";
    }
}

コードの解説

パッケージ宣言: この行は、クラスが mypack パッケージに属していることを示しています。

package mypack;

クラス宣言: クラス Main を宣言しています。

public class Main {

メインメソッド: プログラムのエントリーポイントです。test(null) を呼び出し、その結果を標準出力に出力します。

public static void main(String[] args) {
    System.out.print(test(null));
}

テストメソッド: test メソッドは Object 型の引数 obj を受け取ります。

private static String test(Object obj) {
    try {
        System.out.println(obj.toString());
    } catch (NullPointerException e) {
        return "A";
    } finally {
        System.out.println("B");
    }
    return "C";
}

ここでは、try ブロック内で obj.toString() を呼び出しますが、obj が null のため NullPointerException が発生します。catch (NullPointerException e) ブロックで例外がキャッチされ、「A」を返そうとします。finally ブロックは必ず実行され、「B」が出力されます。catch ブロックが「A」を返すため、その後の return "C"; は実行されません。

実行結果

プログラムの出力は次のようになります：

BA

ここで重要なのは、finally ブロックが return の前に必ず実行されるため、「B」が最初に出力され、その後「A」が出力される点です。
