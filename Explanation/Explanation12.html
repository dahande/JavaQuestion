Javaのメソッドオーバーロード

Javaでは、同じクラス内で同じ名前のメソッドを複数定義することができます。これを「メソッドオーバーロード」と呼びます。メソッドオーバーロードを実現するためには、各メソッドのシグネチャが異なる必要があります。

メソッドシグネチャとは？

メソッドシグネチャは、メソッドの名前と引数リスト（引数の数、型、順序）によって決まります。戻り値の型はシグネチャに含まれません。

オーバーロードの具体例

以下は、シグネチャの違いによるメソッドオーバーロードの具体例です。

public class OverloadExample {
    // 引数なし
    public void print() {
        System.out.println("引数なしメソッド");
    }

    // 引数が1つ（int型）
    public void print(int num) {
        System.out.println("引数が1つ: " + num);
    }

    // 引数が2つ（int型, String型）
    public void print(int num, String str) {
        System.out.println("引数が2つ: " + num + ", " + str);
    }

    // 引数が1つ（String型）
    public void print(String str) {
        System.out.println("引数が1つ: " + str);
    }

    public static void main(String[] args) {
        OverloadExample example = new OverloadExample();
        example.print();                  // 出力: 引数なしメソッド
        example.print(10);                // 出力: 引数が1つ: 10
        example.print(20, "Hello");       // 出力: 引数が2つ: 20, Hello
        example.print("Java");            // 出力: 引数が1つ: Java
    }
}

オーバーロードの条件

メソッドオーバーロードの条件は以下の通りです:

  • メソッドの名前が同じであること
  • 引数リスト（引数の数、型、順序）のいずれかが異なること

戻り値の型やアクセス修飾子はシグネチャに含まれないため、これらが異なるだけではオーバーロードとはなりません。

注意点

メソッドオーバーロードを使用する際の注意点は以下の通りです:

  • オーバーロードされたメソッドの呼び出しがどのメソッドに対応するかは、引数の型や数によって決まります。
  • 引数の型が曖昧な場合、コンパイルエラーが発生することがあります。
